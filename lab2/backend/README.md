### Вариант 4000

Доработать ИС из ЛР1 следующим образом:

1. **Добавить массовый импорт объектов.**  
   - Формат файла согласовать с преподавателем.  
   - Импортируемый файл должен загружаться на сервер через интерфейс веб-приложения.

2. **Реализовать транзакционность импорта.**
   - При возникновении ошибок при импорте не должен быть создан ни один объект.
   - Весь процесс импорта выполняется атомарно.

3. **Проверка пользовательского ввода.**
    - Все данные проверяются в соответствии с ограничениями предметной области из ЛР1.  
   - Ошибки валидации должны приводить к откату транзакции.

4. **Поддержка вложенных объектов.**  
   - При наличии вложенных объектов в основной сущности значения их полей задаются в той же записи импорта.

5. **Интерфейс истории импортов.**  
   - Пользователь видит только свои операции импорта.  
   - Администратор видит все операции.  
   - В истории должны отображаться:
     - ID операции;
     - статус завершения;
     - пользователь, запустивший операцию;
     - число добавленных объектов (для успешно завершённых операций).

6. **Ограничения уникальности.**  
   - Добавить в модель из ЛР1 новые уникальные ограничения, проверяемые **на программном уровне**.  
   - Эти ограничения реализуются в бизнес-логике, а не в БД.

7. **Нагрузочное тестирование (Apache JMeter).**  
   - Смоделировать одновременную работу нескольких пользователей.  
   - Проверить корректность изоляции транзакций.  
   - При необходимости изменить уровень изоляции и обосновать выбор.  
   - Сценарий должен включать:
     - создание, редактирование, удаление и импорт объектов;
     - проверку конфликтов при одновременном редактировании или удалении одного объекта;
     - проверку соблюдения уникальности при одновременном создании одинаковых объектов.
